# Проектная работа "Веб-ларек"

Стек: HTML, SCSS, TS, Webpack

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами
- src/components/base/ — папка с базовым кодом

Важные файлы:
- src/pages/index.html — HTML-файл главной страницы
- src/types/index.ts — файл с типами
- src/index.ts — точка входа приложения
- src/styles/styles.scss — корневой файл стилей
- src/utils/constants.ts — файл с константами
- src/utils/utils.ts — файл с утилитами

## Базовый код
### Класс EventEmitter implements IEvents
Позволяет подписываться на события и уведомлять о наступлении события.
Класс имеет методы ```on``` ,  ```off``` ,  ```emit```  — для подписки на событие, отписки от события и уведомления о наступлении события соответственно.
Также есть методы  ```onAll``` и  ```offAll```  — для подписки на все события и сброса всех подписок.

### Абстрактный класс Model<T>
Служит базовой моделью для других классов.
Имеет метод ```emitChanges(event: string, payload?: object)```, который используется для сообщения всем слушателям об изменениях в модели. Он принимает строку события ```event``` и необязательный объект данных ```payload```, который по умолчанию равен пустому объекту. Метод вызывает метод ```emit``` объекта ```events```, чтобы отправить событие с данными.

### Абстрактный класс Component<T>
Абстрактный класс для работы с DOM-элементами в дочерних компонентах.
Конструктор принимает для чтения HTMLElement:
```protected constructor(protected readonly container: HTMLElement)```.
Класс имеет методы:
- ```toggleClass(element: HTMLElement, className: string, force?: boolean)``` - метод переключения класса
- ```setDisabled(element: HTMLElement, state: boolean)``` - метод смены состояние ```disabled```
- ```protected setHidden(element: HTMLElement)``` - метод для отображения элемента
- ```protected setVisible(element: HTMLElement)``` - метод для скрытия отображения элемента
- ```protected setImage(element: HTMLImageElement, src: string, alt?: string)``` - метод для установки изображения с альтернативным текстом
- ```render(data?: Partial<T>): HTMLElement``` - метод для возврата (отрисовки) элемента с переданными данными

### Класс Api
Класс для подключения к api проекта, имеет методы ```get```, ```post```. Принимает в конструкторе строку базового url для обращения к api и объект дополнительных параметров метода ```fetch```.

### Класс AppState extends Model<IAppState>
Класс наследуется от абстрактного класса ```Model``` для хранения данных приложения: корзины, каталога, заказа. Содержит методы управления приложением и его данными: загрузки/очистки данных корзины, добавления/удаления товара из корзины, методы валидации и оформления заказа.

Имеет поля:
 - ```basket``` представляет корзину товаров и инициализируется пустым массивом
 - ```catalog``` представляет каталог товаров и инициализируется пустым массивом объектов
 - ```loading``` указывает на состояние загрузки и инициализируется логическим значением
 - ```order``` представляет информацию о заказе, такую как email, телефон, оплата, адрес, общая сумма и список товаров
 - ```formErrors``` содержит ошибки формы заказа и инициализируется пустым объектом

Имеет методы:
 - ```toggleOrderedLot(id: string, isIncluded: boolean)``` - переключает выбранный товар в заказе в зависимости от его наличия в списке товаров
 - ```getCount(): number``` - возвращает количество товаров в заказе
 - ```clearBasket()``` - очищает корзину
 - ```getTotal()``` - вычисляет общую сумму заказа 
 - ```setCatalog(items: IProductItem[])``` - устанавливает каталог товаров, преобразуя каждый элемент в ProductItem и уведомляя об изменениях
 - ```setPreview(item: ProductItem)``` - устанавливает выбранный товар для просмотра и отправляет событие об изменении просмотра
  - ```validateOrder``` - метод для валидации полей первой формы оформления заказа и отображения ошибок валидации 
 - ```validateContactOrder``` -  метод для валидации полей второй формы оформления заказа и отображения ошибок валидации 
 - ```setOrderField``` - устанавливает значения полей заказа при заполнении первой формы заказа с выбором способа оплаты и заполнением адреса доставки
 - ```setContactsField``` - устанавливает значения полей заказа при заполнении второй формы заказа с заполнением контактов
 - ```getProduct``` -  метод возвращает массив элементов каталога, добавленных в заказ
 - ```clearOrderOpenFields``` - метод очистки полей заказа адреса доставки и способа оплаты 

 ### Класс ProductItem extends Model<IProductItem>
 Класс наследуется от абстрактного класса ```Model``` для отображения данных приложения, имеет поля товара: id, описание, заголовок, изображение, категория, цена

## Компоненты
### Класс Card extends Component<IProductItem>
Класс наследуется от абстрактного класса ```Component``` для отображения карточек с товарами в каталоге.
Класс содержит защищенные поля с DOM-элементами (заголовок, изображение, кнопка, категория, цена), в конструкторе происходит инициализация карточки товара. Имеет методы ```get``` и ```set``` для получения/установки id, заголовка, описания, изображения. 

### Класс CardProduct extends Card
Класс наследуется от класс ```Card``` для отображения карточки товара и добавления его в корзину. Содержит защищенные поля с DOM-элементами (заголовок, изображение, описание, кнопка, категория, цена, id), в конструкторе происходит инициализация карточки товара. 

### Класс CardBasket extends Component<ICardBasket>
Класс наследуется от абстрактного класса ```Component``` для отображения карточек товаров в списке товаров корзины.
Класс содержит защищенные поля с DOM-элементами (заголовок, индекс, кнопка, цена), в конструкторе происходит инициализация карточки товара. Имеет методы ````set``` для установки индекса, цены, заголовка.  

### Класс Basket extends Component<IBasketView>
Класс наследуется от абстрактного класса ```Component``` для отображения корзины. Класс содержит защищенные поля с DOM-элементами (сумма заказа, список товаров, кнопка), в конструкторе происходит инициализация карточки корзины. Имеет методы ```get``` и ```set``` для установки итоговой стоимости, заполнения списка корзины, для разблокировки кнопки.


### Класс Form<T> extends Component<IFormState> 
Класс наследуется от абстрактного класса ```Component``` для отображения форм приложения. Класс содержит защищенные поля с DOM-элементами (ошибки и кнопка сабмит), конструктор принимает DOM-элемент формы и события, происходит инициализация формы, включая добавление слушателей на инпут и на сабмит, предотвращающий стандартное поведение. 
Имеет методы: 
- ```protected onInputChange(field: keyof T, value: string)```, принимающий поле и значение, внутри которого назначается слушатель на изменение поля, предоставляющий информацию об измененном поле и его значении 
- ```set valid(value: boolean)``` и ```set errors(value: string)``` для отключения кнопки подтверждения и вывода ошибки
- ```render(state: Partial<T> & IFormState)```, принимающий состояние формы и возвращающий форму с учетом полученных данных.   

### Класс Order extends Form<IOrderForm> 
Класс наследуется от класса ```Form``` для отображения первой формы оформления заказа с выбором способа оплаты и вводом адреса доставки. Класс содержит защищенные поля с DOM-элементами (две кнопки соответствующие способу оплаты, инпут для ввода адреса), конструктор принимает DOM-элемент формы и события, происходит инициализация формы, включая добавление слушателей на выбор кнопки и изменение их стилей. Имеет методы ```toggleOrderButton(buttonOn: HTMLButtonElement, buttonOff: HTMLButtonElement)``` для переключения выбранных кнопок и ```clearOrderOpenFields``` для сброса выбранных кнопок и заполнения инпута. 

### Класс OrderSubmit extends Form<IOrderContactForm>
Класс наследуется от класса ```Form``` для отображения второй формы оформления заказа с вводом контактных данных: номера телефона и электронной почты. Класс содержит защищенные поля с DOM-элементами (инпуты для ввода номера телефона и электронной почты), конструктор принимает DOM-элемент формы и события, происходит инициализация формы. Имеет методы ```set``` для установки значения инпутов.  

### Класс Modal extends Component<IModalData>
Классы наследуется от абстрактного класса ```Component``` для отображения модальных окон. Класс содержит защищенные поля с DOM-элементами (элемент модального окна и кнопка закрытия). Конструктор принимает DOM-элемент модалки и события, происходит инициализация модального окна, включая добавление слушателей на кнопку закрытия, на контейнер (вызывающие закрытие) и слушатель на контент модалки, предотвращающий дальнейшее всплытие клика.
Имеет методы:
- ```set content(value: HTMLElement)``` - устанавливает принимаемое значение в модальное окно
- ```open()``` - открывает модальное окно, генерирует событие открытия модального окна
- ```close()``` - закрывает модальное окно, генерирует событие закрытия модального окна
- ```render(data: IModalData): HTMLElement ``` - принимает данные модального окна (DOM-элемент), вызывает родительский метод для отрисовки элемента и метод открытия и возвращает DOM-элемент.

### Класс Page extends Component<IPage>
Класс наследуется от абстрактного класса ```Component``` для отображения основной страницы приложения. Класс содержит защищенные поля с DOM-элементами страницы (элемент корзины, элемент счетчика корзины, элемент основного контента (галерии), элемент блока хэдера). Конструктор принимает DOM-элемент и события, происходит инициализация страницы, включая добавление слушателя на элемент корзины, который генерирует событие открытия корзины.
Имеет методы:
- ```set counter(value: number)``` - устанавливает значение добавленных элементов в корзину на главной странице
- ```set catalog(items: HTMLElement[])``` - принимает массив элементов и заменяет содержимое элемента основного контента (галерии)  
- ```set locked(value: boolean)``` - добавляет или удаляет класс, отвечающий за блокировку прокрутки страницы   

### Класс Success extends Component<ISuccess> 
Класс наследуется от абстрактного класса ```Component``` для отображения окна с уведомлением об успешном оформлении заказа. Класс содержит защищенные поля с DOM-элементами (кнопка закрытия, текст уведомления). Конструктор принимает DOM-элемент и события, происходит инициализация окна уведомления с проверкой на совершения клика и установкой слушателя по проверке условия. Имеет метод ```set``` для уставки текса уведомления с указанием стоимости заказа. 

### Класс LarekApi extends Api implements ILarekApi
Класс наследуются от класса ```Api``` для получения и отправки данных с api проекта. Принимает в конструкторе строку базового url для обращения к api и объект дополнительных параметров метода ```fetch```. Методы класса ```get```и ```post``` возвращают объект с данными, полученными с api.

## Основные типы данных
```
interface IEvents {
    on<T extends object>(event: EventName, callback: (data: T) => void): void;
    emit<T extends object>(event: string, data?: T): void;
    trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void;
}

interface IFormState {
    valid: boolean;
    errors: string[];
}

interface IPage {
    counter: number;
    catalog: HTMLElement[];
    locked: boolean;
}

// интерфейс описания карточки товара
interface ICard<T> {
    image: string;
    title: string;
    category: string;
    price: number | null;
}

// интерфейс описания товара
interface IProductItem<T> {
    id: string;
    description: string | string[];
    image: string;
    title: string;
    category: string;
    price: number | null;
}

interface IModalData {
    content: HTMLElement;
}

// интерфейс описания формы заказа
interface IOrderForm {
    payment: string;
    address: string;
}

export interface IOrderContactForm {
    email: string;
    phone: string;
}

interface IAppState {
    catalog: IProductItem[];
    basket: string[];
    preview: string | null;
    order: IOrder | null;
    loading: boolean;
}
```
## Список основных событий в приложении
 - ```items:changed``` - изменились элементы каталога
 - ```card:select``` - выбрана карточка товара для просмотра
 - ```item:delete``` - удаление товара из корзины
 - ```order:open``` - отрытие первой формы оформления заказа
 - ```order:submit``` - заполнения первой формы оформления заказа, открытие второй формы заказа
 - ```contacts:submit``` - заполнения второй формы оформления заказа, отправка заказа
 - ```preview:changed``` - открытие просмотра выбранного товара
 - ```modal:open``` - открытие модального окна
 - ```modal:close``` - закрытие модального окна
 - ```product:buy``` - добавление товара в корзину
 - ```basket:open``` - открыть корзину
 - ```formErrors.firstStep:change``` - изменение ошибок валидации при заполнении первой формы оформления заказа
 - ```formErrors.secondStep:change``` - изменение ошибок валидации при заполнении второй формы оформления заказа
 - ```order.payment:change``` - изменение способа оплаты
 - ```/^order\..*:change/``` - изменение любого из полей при заполнении первой формы оформления заказа
 - ```/^contacts\..*:change/``` - изменение любого из полей при заполнении второй формы оформления заказа
